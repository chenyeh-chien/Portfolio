### 環境需求

| 項目 | 最低需求 (Minimum) | 建議需求 (Recommended) | 備註 |
| --- | --- | --- | --- |
| CPU 核心 | 8 vCPU | 16 vCPU+ | 必須支援並開啟 Nested Virtualization。 |
| 記憶體 (RAM) | 16 GB | 32 GB ~ 64 GB | Windows Guest 需要分配 4-8G，Nexus 需要 8G+。 |
| 硬碟 (SSD) | 200 GB | 500 GB+ | 必須是 SSD (最好是 NVMe)。HDD 會讓 VM 重置時間增加數倍。 |
| 作業系統 | Ubuntu 22.04 LTS | Ubuntu 22.04 LTS | 目前 CAPEv2 安裝腳本在此版本最穩定。 |

****

### 第一步：準備與安裝 KVM 虛擬化環境
在安裝 CAPE 之前，Ubuntu 必須具備執行虛擬機的能力。CAPE 主要是透過 KVM/QEMU 來驅動 Guest VM。

#### 檢查硬體是否支援虛擬化：
```bash
egrep -c '(vmx|svm)' /proc/cpuinfo
```
如果輸出大於 0，代表支援。

#### 確認 KVM 加速
```bash
sudo apt update && sudo apt install -y cpu-checker
kvm-ok
```

#### 安裝 KVM 核心組件：
```bash
sudo apt install -y qemu-kvm libvirt-daemon-system libvirt-clients bridge-utils virtinst virt-manager
```

#### 啟動並檢查 libvirt 服務：
```bash
sudo systemctl enable --now libvirtd
# 確認運作中
sudo virsh list --all
```
****

### 第二步： 安裝 cape2.sh

#### 下載腳本：  
```bash
wget https://raw.githubusercontent.com/kevoreilly/CAPEv2/master/installer/cape2.sh
```

#### 準備環境 (避免腳本報錯)
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl sudo
```

#### 執行安裝：  
注意：這個過程會安裝 PostgreSQL, Suricata (網路監控), YARA 等工具，大約需要 20-40 分鐘
```bash
chmod +x cape2.sh

# 執行完整安裝 (包含 env, deps, cape 等)
sudo ./cape2.sh all
```

#### 檢查是否安裝成功
```bash
# 檢查 CAPE 核心引擎 (負責任務排程)
sudo systemctl status cape.service

# 檢查 CAPE Web 介面 (負責 API 與 網頁)
sudo systemctl status cape-web.service

# 檢查 CAPE 處理節點
sudo systemctl status cape-processor.service
```

#### 虛擬化網路檢查 (檢查神經)  
CAPE 必須有一條「私有的地下通道」來跟未來的 Windows VM 溝通：
```bash
# 檢查虛擬化服務
sudo virsh net-list --all
```

### 第三步：製作並配置 Guest VM (測試的關鍵)
這是最花時間的一步，因為你需要準備一個「被監控」的作業系統映像檔。  

#### 確認libvirt安裝成功：
```bash
sudo systemctl enable --now libvirtd
sudo systemctl start libvirtd

# 再次檢查指令是否可用
virsh --version

# 重要：權限設定 (不設定 CAPE 會動不了)
sudo usermod -aG libvirt $USER
sudo usermod -aG kvm $USER
```

#### CAPE 需要一個虛擬網段來分配 IP 給沙箱。請輸入：
```bash
sudo virsh net-list --all

# 如果顯示 default 但狀態是 inactive：
sudo virsh net-start default
sudo virsh net-autostart default
```

#### 建立 VM：  
使用 virt-manager (圖形介面) 或指令建立一個 Windows 10 VM。  
```bash
# 檢查 KVM 模組是否已載入
lsmod | grep kvm

# 為了管理方便，我們在 /home/cape 下建立一個專用的存儲池
sudo mkdir -p /home/cape/vms

```

#### 下載 Windows 10 ISO 並存放於 `/temp`:   
https://www.microsoft.com/zh-tw/software-download/windows10ISO

#### 使用virtual machine manager來創立windows 10 VM
1. 安裝windows 10 pro 或 enterprise 版本
2. 安裝flare-vm來建制沙箱環境：
- Flare-VM 特色：
    - 安裝大量分析工具（Wireshark, x64dbg, Ghidra 等）。
    - 預設關閉 Windows Defender、Windows Update 和防火牆（使其易受侵害）。
    - 更改系統設定，使其看起來不像虛擬機。

```powershell
# 1. 強制設定 TLS 1.2
[Net.ServicePointManager]::SecurityProtocol = [System.Net.SecurityProtocolType]::Tls12

# 2. 定義下載網址與儲存路徑
$url = "https://raw.githubusercontent.com/mandiant/flare-vm/master/install.ps1"
$output = "$HOME\Desktop\install.ps1"

# 3. 使用 WebClient 下載
(New-Object System.Net.WebClient).DownloadFile($url, $output)

# 4. 檢查檔案是否存在
ls $output
```

- 下載後檢查：
```powershell
# 解鎖檔案（解除下載保護）
Unblock-File "$HOME\Desktop\install.ps1"

# 開放執行權限
Set-ExecutionPolicy Unrestricted -Force

# 正式執行
cd "$HOME\Desktop"
.\install.ps1
```

3. 更改kvm.conf設定
```bash
sudo nano /opt/CAPEv2/conf/kvm.conf
```
```ini
[kvm]
# 這裡列出你有多少台機器，用逗號隔開
machines = flare_vm_1

[flare_vm_1]
label = FlareVM_Win10  # virt-manager 裡的名字
platform = windows
ip = 192.168.122.10    # 你的 VM IP
snapshot = Ready_To_Analyze  # 你的快照名稱
interface = virbr0
```

#### 把agent送進去:  
1. 在 Ubuntu 主機開啟一個簡單的 Python Web Server：
```bash
cd /opt/CAPEv2/agent/
python3 -m http.server 8080
```

2. 在 Windows VM 裡打開瀏覽器 (Edge)，網址輸入：http://192.168.122.1:8080 (通常 KVM 主機的 IP 是 .1)。

3. 下載 agent.py
4. 安裝 Python 3.10 (32-bit): [Windows installer 32-bit](https://www.python.org/ftp/python/3.10.11/python-3.10.11.exe)
- 執行安裝程式後，請注意以下勾選項目：
    - 不要 勾選 "Add Python to PATH"（這樣才不會跟 FLARE-VM 預設的 3.13 衝突）。
    - 點擊 "Customize installation"。
    - 一路點 Next 到最後一頁 "Advanced Options"。
    - 將 "Customize install location" 修改為一個簡單的路徑，例如：C:\Python310-x86
    - 點擊 Install。

5. 安裝Flask:
```bash
C:\Python310-x86\python.exe -m pip install flask requests
```

6. 新增捷徑run_agent.bat:
- 在桌面新增text取名為run_agent.bat並edit:
```
@echo off
cd /d "C:\你的agent存放路徑"

:: 直接使用絕對路徑呼叫 32 位元的 Python
"C:\Python310-x86\python.exe" agent.py
pause
```
- 右鍵點擊run_agent.bat, 進入properties, 在advanced裡面選擇點擊以系統管理員身分執行
- 將捷徑移至startup資料夾 (在win + R 輸入 shell:startup)
- 雙擊點擊捷徑，應該會出現一個黑色的cmd視窗，代表agent正在執行

7. 驗證連線
```bash
netstat -an | findstr 8000
```

8. 關閉安全機制：
- 手動加入 Exclusions
    1. Open Windows Security (可以在 Start menu 搜尋)。
    2. Click on Virus & threat protection.
    3. Under "Virus & threat protection settings", click Manage settings.
    4. Scroll down to the bottom to the Exclusions section, and click Add or remove exclusions.
    5. Click the + Add an exclusion button and select Folder.
    6. Select your C:\ drive (or specifically C:\Python310-x86 and your Agent folder).

- 徹底關閉 Defender
    1. Press Win + R, type gpedit.msc, and press Enter.
    2. Go to: Computer Configuration > Administrative Templates > Windows Components > Microsoft Defender Antivirus.
    3. On the right, find "Turn off Microsoft Defender Antivirus".
    4. Double-click it, select Enabled, and click OK.
    5. Restart the VM.

#### 使用 Virt-Manager 拍攝「執行中」快照
1. 開啟 Virt-Manager：在 Ubuntu 桌面打開虛擬機管理員。
2. 啟動 cuckoo1：確保 Windows 已經進入桌面，且 Agent 正在跑。
3. 進入快照分頁：
    - 雙擊進入 cuckoo1 的控制視窗。
    - 點擊上方工具列的 「Manage VM snapshots」 圖示（通常是一個像相機或時鐘的圖示）。
4. 新增快照：
    - 點擊左下角的 「+」 號。
    - Name: 輸入 clean_agent_ready (務必跟 kvm.conf 一致)。
    - Description: 隨便寫，例如 Running with agent。
    - 點擊 Finish。
5. 關鍵點：Virt-Manager 會自動處理「記憶體保存」，這就是 CAPE 想要的 "Running" 狀態。

#### 修正ip
1. 找出 Ubuntu 在虛擬網路的真實 IP
請在 Ubuntu 終端機輸入：
```bash
ip addr show virbr0
```
你會看到類似 inet 192.168.122.1/24 的輸出。那個 192.168.122.1 才是正確的地址。

2. 修改 cuckoo.conf
打開設定檔：
```bash
sudo nano /opt/CAPEv2/conf/cuckoo.conf
```

找到 [resultserver] 區塊，修改 ip：
```ini
[resultserver]
# 將原本的 192.168.1.1 改成你剛才查到的 virbr0 IP (通常是 192.168.122.1)
ip = 192.168.122.1
port = 2042
```

#### 重新執行cuckoo.py
```bash
python3 cuckoo.py
```
